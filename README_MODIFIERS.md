# Инструкция по добавлению модификаторов в Supabase

## Как добавить модификаторы для тестирования

### Шаг 1: Откройте Supabase Dashboard
1. Перейдите на https://supabase.com/dashboard
2. Выберите проект `test.coffee_panel-v1`
3. Откройте SQL Editor

### Шаг 2: Выполните SQL скрипт
Скопируйте и выполните содержимое файла `add_modifiers_to_supabase.sql`

Этот скрипт создаст:
- **Для Капучино** (id: `5a83c268-d7de-47cd-9464-85ad086e2266`):
  - Группа "Размер" (S, M, L) - обязательная
  - Группа "Молоко" (Обычное, Соевое, Миндальное, Кокосовое) - опциональная
  - Группа "Дополнительно" (Ваниль, Карамель, Маршмеллоу, +Шот эспрессо, Лёд, Сахар) - опциональная, множественный выбор

- **Для Чай** (id: `b368b7a0-26cb-4751-8746-d30025d215ed`):
  - Группа "Тип чая" (Зелёный, Чёрный, Травяной, Фруктовый) - обязательная
  - Группа "Добавки" (Мёд, Лимон, Имбирь, Мята) - опциональная, множественный выбор

### Шаг 3: Проверьте результат
После выполнения скрипта:
1. Обновите страницу приложения (Ctrl+F5 для очистки кэша)
2. Нажмите на товар "Капучино" или "Чай"
3. Должен открыться экран с кубиками-модификаторами

## Структура данных

### Таблицы:
- `ModifierGroup` - группы модификаторов (Размер, Молоко, Дополнительно)
- `ModifierOption` - опции внутри групп (S, M, L и т.д.)
- `ProductModifierGroup` - связь между продуктами и группами модификаторов

### Поля ModifierGroup:
- `name` - название группы (Размер, Молоко, Дополнительно)
- `isRequired` - обязательная ли группа (true/false)
- `type` - тип выбора: 'SINGLE' (одиночный) или 'MULTIPLE' (множественный)
- `sortOrder` - порядок сортировки

### Поля ModifierOption:
- `name` - название опции (S, M, L, Обычное и т.д.)
- `description` - описание (250 мл, 350 мл и т.д.)
- `price` - дополнительная цена
- `emoji` - эмодзи для отображения (опционально)
- `isActive` - активна ли опция
- `sortOrder` - порядок сортировки
